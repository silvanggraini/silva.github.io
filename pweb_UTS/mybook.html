Mengenal SVG : Format Gambar terbaik Untuk Website

A. Sejarah dan Perkembangan SVG

Vektor grafik sudah dikenal sejak sebelum masa Internet, tetapi tidak sukses untuk diterima dan diterapkan oleh banyak pihak. Raster format lebih dapat diterima, tetapi format ini cenderung proprietary dan membutuhkan tool khusus untuk rendering. Pada raster format data teks tidak dapat di-embeded secara langsung ke dalam file yang bersangkutan. XML dapat digunakan untuk membuat text-based file dengan tidak tergantung dari jenis platformnya. XML juga telah berhasil digunakan untuk mendifinisikan sejumlah Vector Markup Language (VML) dan Precision Graphics Markup Language (PGML). Kemampuan ini mendorong W3C untuk mengembangkan Scalable Vector. Graphic (SVG).

Pada tahun 1999 Scalable vector graphic (SVG) mulai dikembangkan oleh World Wide Web Consortium (W3C) dan beberapa nama besar ikut serta dalam sejarah. Pada tahun 2001, SVG mendapatkan facelift untuk menyertakan profil ponsel (mobile), kemampuan cetak (print) dan meningkatkan pengaturan display. Dan pada tahun itu pula tanggal 4 September SVG versi 1.0 mulai dirilis. Pada 14 Januari 2003 SVG 1.1 dirilis dan pada 16 Agustus 2011 SVG 1.1 (versi dua ) dirilis.

Satu hal yang menarik dari perkembangan SVG pada awalnya adalah banyak desainer web yang tidak tertarik dengan format ini karena tidak di support oleh beberapa major web browser, sedangkan browser lain sudah menggunakan SVG, namun untuk saat ini SVG sudah mulai populer, karena para web developer sudah mulai menyadari benefit atau keunggulan dari SVG untuk kebutuhan website. Sekarang sejumlah peramban web populer seperti Mozilla Firefox, Internet Explorer, Google Chrome, Opera, dan Safari sudah mulai mendukung penggunakan format SVG.

SVG (Scalable Vector Graphics) adalah format gambar yang menggunakan XML (Extensible Markup Language) sebagai dasar untuk membentuk gambar vektor dua dimensi. Grafik ini dapat terdiri dari garis, gambar dan teks yang dapat diperbesar atau diperkecil tanpa mengubah kualitasnya.

Sebuah gambar dengan format SVG disimpan dalam bentuk file XML. Ini berarti, SVG dapat dicari, di-index, ditulis dengan bahasa pemograman dan dikompres. Karena SVG dirancang untuk menggunakan bahasa XML (Extensible Markup Language), maka kita bisa mengubahnya dengan aplikasi teks apa saja seperti pada web browser, teks editor dan software desain grafis berbasis vektor. SVG telah dirancang untuk dapat diintegrasi dengan berbagai Web standard seperti XLink, XML NameSpace, CSS, dan XSL. Dengan kata lain SVG adalah format yang independen dan tidak proprietary.
